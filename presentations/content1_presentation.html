Search Basics Review

Basic Search Syntax:

index=main error
  • Keywords: Simple text search (case-insensitive)
  • Index: Specifies which index to search
  • Time Range: Defaults to last 24 hours
  • Pipe (|): Chains commands together

Boolean Operators

AND (implicit and explicit):

error failed error AND failed

OR:

error OR warning OR critical

NOT:

error NOT warning
Note: Boolean operators must be UPPERCASE. Order of precedence: NOT, AND, OR

Wildcards

Asterisk (*) - matches any number of characters:

fail* *error error*message
Warning: Leading wildcards (*error) are slow! Avoid when possible.

Best Practices:

  • Use wildcards at the end of terms
  • Be as specific as possible
  • Consider using field filters instead

Field Extraction

Automatic Fields:

  • host, source, sourcetype
  • _time, _raw, _indextime
  • Extracted at search time based on sourcetype

Field Filtering:

index=web status=404 index=web status>=400 status<500 index=web user=admin OR user=root

Fields Command

Display specific fields:

index=web | fields host, source, status, response_time

Remove fields from display:

index=web | fields - _raw, _indextime
Tip: Use fields early to reduce data volume and improve performance

Where Command

Filter results with boolean logic:

index=web | where status=404 index=web | where status > 400 AND status < 500 index=web | where isnotnull(user_id)

where vs field=value:

  • field=value: Earlier in pipeline, index-time filtering
  • where: After pipe, search-time filtering with complex logic

Stats Command Introduction

Basic aggregation:

index=web | stats count index=web | stats count by status index=web | stats count, avg(response_time) by host

Common statistical functions:

  • count, sum, avg, min, max
  • distinct_count (dc)
  • values (unique values)
  • list (all values)

Top and Rare Commands

Top command - most common values:

index=web | top limit=10 user index=web | top status, host

Rare command - least common values:

index=web | rare limit=10 src_ip index=web | rare user, action

Search Best Practices

  1. Be Specific: Use index, sourcetype, host filters
  2. Time Range: Limit to necessary time window
  3. Filter Early: Use field filters before pipes
  4. Avoid Leading Wildcards: Don't use *error
  5. Use Fields Command: Reduce data volume early
  6. Leverage Field Filters: field=value is faster than keywords

Search Performance Tips

# Bad: Broad search error # Better: More specific index=main error # Best: Very specific with time range index=main earliest=-1h sourcetype=access_combined status=500
Remember: Every character matters for search performance!

SPL Search Pipeline

Search pipeline flows left to right:

index=web status>=400 | stats count by status | where count > 100 | sort -count

Pipeline stages:

  1. Retrieve: Get events from index
  2. Process: Transform with commands
  3. Display: Present results

Common Commands Quick Reference

Command Purpose Example
fields Select/remove fields fields host, status
where Filter with boolean where status=200
stats Calculate statistics stats count by host
sort Order results sort -count
top Most common values top 10 user
